Application Overview

The EIOS Article Management Application is a comprehensive system for monitoring, analyzing, and managing health emergency intelligence from the WHO's EIOS (Epidemic Intelligence from Open Sources) platform.

What it does:

  • Fetches Articles: Automatically retrieves articles from the EIOS API based on configured tags and criteria
  • AI Analysis: Uses advanced AI models (OpenAI GPT-4, DeepSeek, or local LLMs) to assess each article for emergency signals
  • Risk Assessment: Evaluates articles for potential acute risks to human health and categorizes threats
  • Classification: Automatically identifies countries, hazards, and risk levels from article content
  • Management Interface: Provides intuitive tools for reviewing, filtering, flagging, and exporting articles

Key Benefits:

  • Automated Monitoring: Continuous background processing of new articles
  • Intelligent Filtering: AI-powered identification of true emergency signals vs. routine news
  • Global Coverage: Monitors health threats across all countries and regions
  • Flexible Export: CSV export capabilities for further analysis and reporting
  • Multi-timezone Support: Handles datetime filtering across different timezones

Key Features

AI-Powered Analysis

Automatically evaluates articles using configurable AI models to identify genuine health emergency signals and assess risk levels.

Advanced Filtering

Multi-criteria filtering system supporting article status, signal classification, countries, hazards, and date ranges with timezone support.

Comprehensive Search

Keyword search across article titles, summaries, AI assessments, countries, hazards, and all content fields.

Batch Operations

Bulk actions for flagging, discarding, and exporting multiple articles simultaneously to streamline workflow.

Analytics Dashboard

Statistical overview with counts by status and signal type, plus top countries and hazards analysis.

Data Export

CSV export functionality with all article data, AI assessments, and metadata for external analysis.

Typical Workflow

1

Configure System

Set up EIOS tags, AI provider credentials, and risk evaluation prompts in the Configuration panel.

2

Fetch Articles

Use "Fetch Articles" button or enable the scheduler for automatic hourly fetching from EIOS.

3

Review Articles

Browse articles, filter by "True Signal" to see AI-identified emergency signals, and review AI assessments.

4

Flag Important Articles

Use "Flag Selected" to mark articles that require follow-up or further investigation.

5

Clean Up

Use "Discard Non-Flagged" to clean up processed articles, keeping only flagged items for ongoing monitoring.

6

Export Data

Export filtered results to CSV for reporting, analysis, or integration with other systems.

Filtering & Search

Filter Articles

Multi-select dropdown with AND logic. Combine multiple criteria:

  • Pinned/Unpinned: Articles marked as important in EIOS
  • Flagged/Unflagged: Articles you've manually flagged for review
  • True Signal/Not Signal: AI-assessed emergency signals

Search

Keyword search across multiple fields:

  • Article titles and summaries (original and translated)
  • AI risk assessments and justifications
  • Extracted countries and hazards
  • All article content and metadata

Case-insensitive partial matching (e.g., "cycl" matches "Cyclone")

Countries & Hazards

Dynamic filters showing countries and hazards found in currently filtered articles. Select multiple options to narrow results further.

Date & Time Range

Filter by AI processing date (not publication date). Supports hourly precision and automatically handles timezone conversion to UTC.

Article Actions

Selection Actions

  • Select All: Select all articles on current page
  • Unselect All: Clear all selections

Batch Actions

  • Flag Selected: Mark for review/follow-up
  • Discard Selected: Mark as processed (not deleted)
  • Export Selected: Download as CSV

Bulk Operations

  • Discard Non-Flagged: Clean up all unflagged articles
  • Export All: Download all filtered results

Article Status

  • New: Recently processed articles
  • Flagged: Marked for review
  • Discarded: Processed and archived

Configuration

EIOS Settings

  • Tags: Comma-separated list of EIOS tags to fetch (e.g., "ephem emro, other tag")

AI Provider Configuration

OpenAI

Configure API key, base URL, and model (e.g., gpt-4, gpt-3.5-turbo)

DeepSeek

Alternative AI provider with own API credentials

Local LLM

Connect to local language models (e.g., Ollama, LM Studio)

Risk Evaluation Prompt

Customize how the AI evaluates articles. Use {text} placeholder where article content should be inserted.

Database Cleanup

Remove old processed articles by date to manage database size. Includes preview functionality before deletion.

Scheduler

Enable automatic hourly fetching and processing of new articles from EIOS. Can be started/stopped from the main interface.
